Governance · v3

Contribute. Stay sharp.

Great Expectations is a disruptive and complete planning engine. Contributions that accidentally turn it back into a polite, average planner will be rejected. Read this first.

01 — Align

Philosophy you must hold.

Anti-consensus and completeness are both non-negotiable. Completeness serves disruption — never water a radical path back into a safe one to clear a check.

02 — Prove

Evidence over assertion.

Any change to the prompt must be backed by a real run — an example in docs/examples/ or your own. No theoretical edits.

03 — Gate

CI must stay green.

Keep changes honest and aligned with the philosophy above. That is the floor, not the ceiling.

How to change the skill

Edit SKILL.md without diluting it.

  1. Fork and branch (feat/... or fix/...).
  2. Edit SKILL.md.
  3. Run at least one back-test with your changed prompt against an existing docs/examples/*.md case (or a new one). Capture before/after.
  4. In your PR, paste the before/after output and the delta in the acceptance metrics — disruption rate, completeness pass rate, falsifiable count.
  5. Open a PR against main using the template.

Examples

Adding or updating demonstration runs.

  • Place under docs/examples/, named topic.md (kebab-case).
  • Include the standard three-field input (Goal / Biggest consensus / Scarcest resource) and the eight output sections from SKILL.md.
  • Keep it honest — these are demonstration runs, not fictional marketing.
  • Update docs/demo.html's sample-run section if you want it featured.

Red lines

Style constraints you may not relax.

  • Frontmatter (name, description, color, author) must stay valid YAML.
  • The banned vocabulary list may be extended, never shrunk.
  • The eight-section output format is mandatory; do not merge or drop sections.
  • The quantitative acceptance targets (disruption ≥50%, completeness >80%, falsifiable ≥3, PESTEL zero gaps) may only be raised, never lowered.
  • Step 3.5 Completeness Mapping with its four penetration tests is the core gate — never removable.

The site

Swiss discipline for docs/.

  • Black/white, single accent (#E30613 <5%), 12-column grid, 8px baseline. No gradients, glass, or large radii.
  • Positive/Negative theme must both remain WCAG AA (contrast ≥ 4.5:1).
  • Zero external runtime requests. System fonts only.

Definition of Done

Mergeable when all hold.

  • SKILL.md still passes every style red line above.
  • At least one example run shows no regression in the metrics.
  • Site (if touched) keeps the Swiss red lines below.
  • PR description includes the metric delta.
