> 一句话定位：一个强制 AI **打破常规、且系统完备** 的宏大规划引擎 Skill。
> 核心理念：**Breaking the average** —— 凡是被多数人认可的路径，已经失去了超额价值。

## 2. 产品定义（Target）

**Great Expectations Skill** 是一个输⼊「宏大目标」、输出「反共识且完备」战略方案的 AI 技能。

- **输入**：目标领域 / 当前最大行业共识 / 最稀缺资源（结构化）
- **输出**：假设清单 → 第一性原理拆解 → 破坏性路径 → 完备性映射 → 脆弱性推演 → 认知偏见审计 → 可证伪断言

它**不是**通用聊天机器人，而是一条**硬编码多阶段工作流 + 负向约束（禁用词库、强制具体化）**的确定性管道。

## 3. 设计骨架（取自 V3，当前最优版）

### 3.1 四大理论支柱（学术地基）
1. **突破性创新战略管理框架** —— 同时覆盖「探索 / 分析 / 综合」三层
2. **破坏性创新理论（价值系统四维度）** —— 价值主张 / 创造 / 传递 / 实现
3. **第一性原理思维 + Kahneman 双系统** —— 强制激活 System 2，抑制 System 1 直觉
4. **BANI 框架** —— 在「非线性环境」中运作，而非「线性 Business-as-Usual」

### 3.2 完备性三维标准
| 维度 | 学术依据 | 核心追问 |
|------|----------|----------|
| 要素穷举性（横向） | MECE + 利益相关者理论 | 动了谁的奶酪？谁被意外赋权？非人类要素如何反噬？ |
| 因果闭环性（纵向） | 系统动力学反馈回路 | A 解决 B，是否在 C 制造更大问题并倒灌摧毁 A？ |
| 时间自洽性（时间轴） | 路径依赖 + 实物期权 | T=1/3/5 时前置决策是否锁死后置选择权？是否保留战略冗余？ |

### 3.3 强制工作流（不可跳过）
```
解构 → 生成(≥2条非线性路径) → [3.5 完备性映射] → 脆弱性推演(BANI) → 认知审计 → 可证伪断言
```
**步骤 3.5（完备性穿透检查）** 是 V3 的灵魂，包含 4 个硬性测试：
1. 利益相关者全息图（谁失权？谁获意外权？）
2. 资源反脆弱校验（资金 / 政策 / 人 / 数据四类，去掉一类还能转吗？）
3. 二阶 / 三阶效应推演（对手怎么动？监管怎么变？）
4. 时空冗余校验（最不可逆时间点在哪？配套 3 个战略期权 Plan B/C）

## 5. 分阶段开发路线图

### Phase 0 — 需求与理论对齐（0.5 天）
- 复盘 `对话参考.md`，确认采用 **V3 为基线**
- 锁定四大理论支柱引用与完备性三维标准
- 定义 D1–D5 模板结构
- **里程碑**：`开发计划.md` 通过评审（即本文）

### Phase 1 — MVP 内核（1 天）【对应 V1】
- 编写最小可用 `SKILL.md`：角色（逆向未来学家 + 第一性原理破坏者）+ 启动语 + 基础执行协议
- 跑通 1 个真实案例，验证「能打破」
- **验收**：输出让你产生"生理性不适/这太激进了"的感觉

### Phase 2 — 学术加固（1.5 天）【对应 V2】
- 注入四大理论支柱为硬约束
- 落地强制输出结构（六模块）与负向约束（禁用词库 + 强制具体化）
- **验收**：假设颠覆率 ≥ 50%；可证伪断言 ≥ 3 条

### Phase 3 — 完备性引擎（2 天）【对应 V3】
- 硬编码步骤 3.5 的 4 个穿透测试
- 新增「完备性压力测试报告」独立章节
- 强调"完备性服务于颠覆，不阉割颠覆"
- **验收**：横向六维度(PESTEL)红队无漏项；资源反脆弱校验通过

### Phase 4 — 量化验收与红队（1.5 天）
- 编写 `evaluation.md`（指标 + 打分表）
- 历史回溯测试：用 V3 复盘 Google Glass 等，验证能否精准指出"缺完备性"的环节
- 反向压力测试：随机抽掉 20% 初始条件，检查是否仍能输出 B/C 计划
- **验收**：完备性压力测试通过率 > 80%

### Phase 5 — 打磨与发布（1 天）
- 重写 `README.md`、补充 `examples/`
- SKILL.md 最终润色（语气、边界、安全护栏）
- **里程碑**：Skill 可在 WorkBuddy / Custom GPT / Claude Project 中一键装载运行

**合计估算：约 7.5 个工作日**

## 6. 量化验收指标（Verification）

| 指标 | 定义 | 目标值 |
|------|------|--------|
| 假设颠覆率 | 被颠覆假设 / 总假设 | ≥ 50% |
| 非线性度 | 非线性命题数 / (线性+非线性) | 越高越好，设基线对比 |
| 完备性通过率 | 通过 3.5 四测试的模块占比 | > 80% |
| 可证伪断言 | 含可被事实证伪断言的数量 | ≥ 3 |
| 横向覆盖度 | PESTEL 六维度无遗漏 | 0 缺项 |
| 时间盲区 | 是否存在"时间黑洞" | 0 处 |

**主观金标准**：看完方案后背发凉、觉得"太激进太得罪人" = 成功；觉得"很有道理" = 失败（只是把旧知识漂亮整理了一遍）。

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| AI 滑向"伟光正套话" | 颠覆性失效 | 负向约束 + 禁用词库 + 强制具体化追问 |
| 完备性阉割颠覆性 | 变回平庸 | 指令明确"完备性服务颠覆"，作废路径才改回平庸 |
| 伦理/安全对齐拒绝打破 | 卡壳 | 角色剥离法；仅拒绝物理定律级红线，其余视为待破牢笼 |
| 输出过长不可控 | 不可用 | 结构化模板 + 强制章节上限 |
| 缺真实案例导致无法验收 | 无从迭代 | Phase 4 强制历史回溯测试 |
